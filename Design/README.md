## 当前核心方案

Kei 是学习型机械 AI 角色，成长分为两条路线：

- 学习路线：从世界、NPC、战斗样本中学习，获得生活数据和战斗数据。
- 扩展路线：通过协议槽挂载战斗数据 CD 或解析协议 CD，获得可切换能力。

三维方案已经固定：

- 饱食度表现为电量，初始值 120。
- San 表现为数据稳定性，初始值 240。
- 血量表现为机体完整度，初始值 180。
- 普通食物吸收率为 20%。

协议槽方案已经固定：

- Kei 开局 15 个物品栏格子。
- 最左侧 7 格为协议单元。
- 初始解锁 1 个协议槽。
- 后续每次解锁 2 个，最终最多 7 个。
- Mk1、Mk2、Mk3 通过右键使用分别解锁后续槽位。
- 生活质量 CD 不进入协议槽。
- 战斗数据 CD 和解析协议 CD 可以进入协议槽。

战斗数据方案当前固定为：

- 数据记录器部署后不阻挡、不碰撞，可以放置在任意地点。
- 空白 CD 可绑定记录器范围内拥有 `epic` 标签的巨兽。
- 已绑定 CD 提交给记录器时，绑定巨兽必须仍在领域范围内。
- 提交成功后开启力场并开始记录。
- 记录中玩家可右键停止，关闭力场并返还空白 CD。
- 绑定巨兽死亡后记录完成，玩家点击记录器收获战斗数据 CD。
- 第一批可记录巨兽为 `deerclops`、`bearger`、`dragonfly`。
- 完成记录后的战斗数据 CD 可直接作为战斗协议插入协议槽。

解析协议方案当前固定为：

- 解析 CD 不视为普通装备。
- 协议槽不视为普通装备栏。
- 头部和身体解析 CD 插槽后，后台生成虚拟装备并装备到隐藏虚拟装备栏。
- 头部和身体虚拟装备进入 `inventory.equipslots`，尽可能继承原装备组件、标签、回调和附加能力。
- 头部和身体虚拟装备防御按原版最高吸收值逻辑结算。
- 手部解析 CD 不生成虚拟装备，只读取攻击力、位面伤害和移速倍率，作为可叠加属性协议加到 Kei 身上。
